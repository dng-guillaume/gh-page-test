Sample

New Features

#11422 Provide the lineage at the sample level for any containers (plate/vial)

  • When selecting a well in the plate viewer (plate or rack), you can click on Sample Lineage to display the lineage of the selected samples.

#11435 Retrieve annotations and control layout from registration

  • The annotations (text, number, controlled vocabulary) of the container and the control layout can be retrieved during the registration.
  • The control layout is displayed on the plate viewer and can be modified by editing the plate.
  • The annotations are displayed on a report and their values can be modified.

Improvements

#11419 Update of the barcode administration accessibility

  • Now, as a Sample user, you have the right to dispose one or multiple containers/racks using the Utilities panel.
  • A new role has been created allowing the barcodes of containers/racks to be changed.
  • A new role has been created to allow containers/racks to be retired. Retired means that the container/rack is renamed (with the suffix RETIRED), moved to the garbage and the validation of the containers/rack is run.

#11209 Improve reception from registration

  • Now, you can go directly from the summary of registration to reception requesting with no need to re-select the containers.

Bugs resolved

#11240 Should not be possible to register a 384 wells plate as a 96

  • It was previously possible to register plates with well positions that did not match the selected format. Such plates will now be marked as invalid at the registration.

#11838 Request validated even if the rack is empty

  • Empty racks (meaning rack with no vial or where all the vials are empty) used to be considered as valid at the requesting step of a job. A new check has been implemented, and they are now marked as invalid.

#11278 Requesting step does not warn when a requested item is invalid

  • For some workflows at the Requesting step, non existing items were considered valid. This is not the case anymore and they are properly marked as invalid.

Bugs resolved

#11904 Plate Mapping management in Execution mode => Error

  • In run mode, at the Plate Mapping step, a control layout and/or a dose layout can be modified in order to apply the correct layout to my analysis. This action used to generate an error. The underlying code has been reworked to fix this issue.

#11881 OML Normalizations breaks when going to the administration

  • In Assay, it is possible to create normalizations that make use of Oracle Machine Learning (OML) behind the scenes. Viewing and editing such normalizations in the administration used to lead to inconsistencies, errors or misconfigurations. The affected pages have been fixed, and these actions can now be carried out without any issues.

#12086 Hit Selection buttons are hidden when they should not

  • At the Sample Analysis step, during the hit selection, if the number of hits was a multiple of 10, the related action buttons were hidden. This is now fixed and the buttons are always displayed as long as you have selected at least one hit.
